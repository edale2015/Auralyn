# Auralyn — HIPAA/FDA Medical Triage Platform (Multi-Tenant SaaS)

### Overview
Auralyn is an AI-powered medical triage platform for flu-like symptoms, leveraging WhatsApp for initial patient assessments. It provides diagnoses and treatment plans for physician review, automates patient communication, and enhances healthcare efficiency and access. The system is designed for continuous improvement through a self-developing AI architecture, with a vision to transform medical triage into a more efficient, patient-centric process. Key capabilities include a unified EHR write architecture, advanced diagnostic engines, and comprehensive security and compliance features. The platform also features extensive command center dashboards for monitoring, predictive analytics, and digital twin simulations, supporting multi-hospital and regional deployments.

### User Preferences
Preferred communication style: Simple, everyday language.

### System Architecture

#### Core Architecture
The system employs a constrained agent architecture with a plan/act/observe loop and a multi-system triage pipeline. It features a unified sheets registry for data configuration, a clinical state builder, and a modular, skill-based orchestration layer. An intelligence layer provides explainability and failure-driven rule suggestions, with an extended learning loop for continuous improvement. The Clinical Intelligence Planning Layer (CIPL) and Clinical Governance & Deployment Layer ensure continuous improvement and safe deployment of clinical changes. The system adheres to a 12-layer architecture encompassing Interface, Normalization, State, Knowledge, Safety, Reasoning, Decision, Learning, Analytics, Governance, Integration, and Orchestration. All clinical EHR writes flow through a single, guarded pipeline ensuring authentication, scope checks, and audit logging. A robust validation system, including golden cases, adversarial generators, and calibration monitors, enforces strict deployment gates. The system incorporates a Medical Command and Control (MCP) Nervous System, allowing for dynamic registration and execution of clinical tools and workflows, with an immutable audit hash chain.

#### UI/UX Decisions
The frontend is built with React 18, TypeScript, `shadcn/ui`, and Tailwind CSS, offering intuitive interfaces for physicians, patients, and administrators. Key dashboards include: Clinical Simulation Lab, Clinical Control Tower, Executive Dashboard, Stress Test Dashboard, Patient Queue Dashboard, FDA Validation Dashboard, Decision Tree Explorer, Live Clinic Console, Production Readiness Console, Clinical QA Dashboard, Clinical Improvement Lab, Care Pathway Optimizer, Skill Graph, Revenue War Room, Governance Command Center, Physician Command Strip, Clinical Brain Dashboard, System Validation Dashboard, Clinical Operations Center, Clinical Brain Page, Agent System Page, Cognitive Brain Page, Communication Dashboard, Master Control, Hospital Dashboard, Live Patient Monitor, and Command Center v2/v3/v4.

#### Technical Implementations
The backend uses Express 5, Node.js, and TypeScript, providing REST API endpoints. It incorporates Centor score calculation, red flag detection, a supervisor gate, and robust LLM integrations. Features include: Clinical Brain Engine (rewritten to v3.0 phase-parallel), Self-Developing Medical AI, Telemedicine Reasoning Assistant, and an Agent System for patient flow. Clinical capabilities include advanced triage logic, medication safety layers, FHIR-lite structured output, Clinical Knowledge Graph, 30+ Complaint Packs with a Visual Rule Builder, Adaptive Control Loop, Case Memory Engine, Unified Clinical Pipeline, Autonomous Operator System, Engine Control Center, Auto-Debug Engine, Agent Coordinator, SMS/WhatsApp Service, Stress Test System, RPA Browser Agent, FDA Submission Package, Live Patient Queue, Predictive Failure Engine, Autonomous Loop, Safety Gate, Immutable Audit Logger, and Explainability integrated into the orchestrator. The system uses a Multi-Agent Debate Engine, Continuous Learning Pipeline with RLHF, and Policy Evolution Engine. All clinical decisions are managed via Postgres KB tables. Core intelligence utilities include Fisher Information Matrix, Natural Gradient, Bayesian Updater, and Hallucination Guards. Security is hardened with AES-256-GCM encryption for credentials, HMAC-SHA validation for Twilio webhooks, and comprehensive PHI guarding before external API calls. Multi-tenant isolation is enforced at various layers. A deep agent Python sidecar provides advanced AI capabilities like research, code review, and workflow upgrades, integrated via a TypeScript bridge. Communication intelligence engines detect patient tone, generate communication scripts, and manage antibiotic demand responses. The system features a unified EHR adapter interface for ECW, Athena, and Epic, with a 5-tier fallback write chain. It also includes an Agent Scope Engine (ASE) for risk-based, role-aware action governance.

#### System Design Choices
Data management uses Firebase Firestore, SQLite, and PostgreSQL, with PHI retention policies. Authentication involves password-only, session-based HMAC for physicians and token-based access for patients, with JWT-based role authentication. Security and quality hardening include bcrypt, JWT security, rate limiting, and PHI Sanitizer. A Global SRE + Resilience Layer provides geo-aware routing, SLA monitoring, automatic debugging, and chaos engineering. Autonomous Governance includes an agent registry, audit agent, incident commander, digital twin, and predictive engine. The Autonomous Operator System is an AI-powered form automation engine. A Template Studio allows visual template editing. The Replay Inspector audits automation runs. A Robotics Control Module manages medical device orchestration. An Autonomous Learning Console provides a unified dashboard for self-testing, self-learning, and governance. The Multi-Patient Command Grid provides a three-pane, hospital-style dashboard with risk-sorted patient grids. The system is designed for multi-region deployment with auto-scaling capabilities and a unified control API. It includes a medical knowledge graph, DAG visualizer, and YAML pipeline engine for complex workflow execution.
