Step 6) Modifiers engine (using CARDS_MODIFIER_MASTER)

Module: services/modifiers.ts

Inputs:

Modifier_Set_ID from router (e.g., MODSET_URGENTCARE_CORE_v1)

FHIR prefill + derived flags

Logic:

pull modifiers (by set)

for each modifier:

if prefilled: ask as confirmation

if missing: ask normally

write answers to CaseState.modifiers

Also compute:

RiskScore from Severity_Weight

TriageUpgradeTarget from modifiers that demand it

✅ Output: consistent modifier answers + risk flags for downstream rules.

Step 7) RULESTRIGGERS execution engine (this is the key glue)

Module: services/rulesEngine.ts

Inputs:

current CaseState (complaint, modifiers, answered secondary questions so far)

RULESTRIGGERS rows filtered by System+Chief_Complaint (+ optional cluster)

Outputs:

actions[] in priority order:

ADD_BUNDLE (adds a GLOBAL_SECONDARY.Bundle_ID to question queue)

SET_CLUSTER (adds/sets candidate cluster)

TRIAGE_UPGRADE (ER/UC/PC)

FLAG_DX (Diagnosis_ID / Dx_Key candidates)

MED_CONTRA_FLAG (tag a med group as blocked)

This engine runs:

after modifiers (Trigger_Level = MODIFIER_GATE)

during question flow (Trigger_Level = SYMPTOM_GATE)

immediately on red flags (Trigger_Level = RED_FLAG_GATE)

✅ Output: dynamic question branching + triage escalation without hardcoding.

Step 8) Secondary question queue builder (GLOBAL_SECONDARY)

Module: services/questionQueue.ts

Inputs:

Router Primary_Secondary_Bundle_ID

GLOBAL_SECONDARY rows

Logic:

✅ Output: deterministic question order with branching.

Step 9) Red flag supervisor gate (your “hard safety shell”)

Module: services/supervisor.ts

Inputs:

Logic:

set disposition to escalation target

✅ Output: safe outputs and audit trail.

Step 10) Cluster selection + disposition engine

This is where you asked for “cluster_selection_disposition.”

Module: services/clusterDisposition.ts

Inputs:

CaseState

Candidate clusters from:

router default cluster

NEURO_GLOBAL_TRIAGE (when System=NEURO)

RULESTRIGGERS SET_CLUSTER

dx master matching (optional)

Algorithm (simple but powerful):

Start with ActiveClusters = {Default_Cluster}

Add clusters from triage maps + rules

Score clusters:

points for matching symptom patterns (you already store diagnosis patterns and criteria)

points for matching modifiers (Key Modifiers)

points for red-flag coherence

Choose top 1–3 clusters

Set default disposition:

from cluster triage map if available

else from diagnosis master thresholds

overridden by supervisor

✅ Output: ActiveClusters, DispositionCandidate, WorkupSuggestions.

Step 11) Medication suggestion engine (the 2-join pipeline)

Module: services/medSuggestions.ts

Join A: Clusters → MedGroups

Use GLOBAL_STANDARDIZED_MEDGROUPS:

filter med groups where Primary_Indications_Clusters contains any ActiveCluster

keep first-line groups first

Apply constraints

From modifiers + FHIR:

pregnancy-related flags

CKD/hepatic flags

allergy class flags

interaction flags (e.g., anticoag + NSAIDs caution)

Use:

GLOBAL_STANDARDIZED_MEDGROUPS.Key_Contraindications

Key_Interactions

renal/hepatic adjust flags

Join B: MedGroups → specific meds

Use ID_MEDICATIONS_MASTER:

join by Medication_Group

filter contraindications + pregnancy + interactions

pick top 1–3 meds

output dose suggestions

✅ Output: “Top options” + “why safe/unsafe” + “what to monitor.”

✅ Output: a ranked diagnosis shortlist and the clinical “why.”
