1) Add a “Change my topic” command (menu reset)

Patients will pick the wrong option sometimes. Give them an escape hatch.

Behavior

If they text any of: menu, change, restart, switch, topic

set __menu.awaitingChoice = true

send the menu again

do not generate a new link+code yet (only after they choose)

This reduces junk encounters and makes the system feel intentional.

2) Add a “status” command (link re-send without new encounter)

Common issue: they lose the link/code.

Behavior

If they text link or code:

if there’s an active encounter with unexpired token/code → resend

else → start fresh

This cuts repeated re-registrations.
