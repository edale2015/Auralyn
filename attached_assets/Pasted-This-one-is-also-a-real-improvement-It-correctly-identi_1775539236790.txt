This one is also a real improvement. It correctly identifies the big structural problems in the old auth path: the token-shape mismatch, the admin 403 bug, split secret resolution, the production-reachable dev fallback, and the duplicate role types. The rewrite fixes those by centralizing verification in verifyAccessToken, using AuthRole as the shared role source, and letting RBAC determine physician-level access instead of relying on a phantom decoded.physician field.

I still would not call it final-hardening yet. A few gaps remain:

1. No runtime validation of the decoded JWT payload
verifyAccessToken() returns jwt.verify(...) as AuthTokenPayload, which is a TypeScript promise, not a runtime guarantee. A token signed with the right secret but containing malformed claims could still get cast into req.user. Add a schema check after verification so id, role, and optional clinicId are actually validated before use.

2. Algorithm / issuer / audience are not pinned
The rewrite signs with jwt.sign(..., { expiresIn: "12h" }), but it does not explicitly set or verify algorithm, issuer, or audience. For a clinical system, I would pin those instead of leaving the contract loose and implicit.

3. Secret validation happens too late
getJwtSecret() throws only when auth code is exercised. That means a bad deployment can still boot and then fail at request time. Better pattern: validate and cache the secret during server startup, so misconfiguration explodes immediately instead of halfway through a clinic day. The current version removes the dangerous fallback, which is good, but it still fails at runtime rather than fail-fast at boot.

4. The RBAC check is slightly redundant
This line:

rbacService.can(decoded.role, "clinical:run") ||
rbacService.can(decoded.role, "*")

does not need the second half if can() already treats "*" in the stored permission set as universal access. Admin already passes can(role, "clinical:run") because admin has ["*"]. The extra check is harmless, but it is a tiny code-smell pebble in the shoe.

5. Token contents are broader than they need to be
The access token currently includes email as part of the bearer payload. That works, but for security minimization I would usually keep JWT claims leaner: sub, role, clinicId, jti, maybe sessionVersion, and fetch email server-side if needed. Less PHI-ish surface drifting around in bearer tokens is usually the tidier lane.

6. No revocation / session invalidation story yet
This rewrite fixes verification consistency, but not token invalidation. If you need forced logout, role downgrade, emergency offboarding, or incident response, you still need a jti or token version check against server-side state. Right now the model is still “valid until expiry if the signature is good.” The file’s tests cover valid, expired, forged, and role-based denial, but not revocation.

7. The tests may still be brittle depending on how ENV is loaded
The test block sets process.env.JWT_SECRET, while the rewritten auth path reads from ENV.JWT_SECRET. If ENV snapshots environment variables at import time, these tests can become sneaky little mirages unless the env is set before module import or ENV is mocked properly.

My verdict: yes, this is directionally strong and materially safer than the original, especially because it fixes the admin lockout and eliminates the split-secret logic. But before I’d stamp it “production-ready,” I would add: payload schema validation, issuer/audience/algorithm pinning, startup-time secret validation, token revocation support, and slimmer JWT claims. The castle is standing now; I’d still add the iron hinges.

3. Replace requirePhysician with a generic permission gate

The uploaded rewrite correctly points out that the old middleware was checking for fields that were never issued and that admin was wrongly blocked from physician routes. // server/auth/rbacService.ts
import type { AuthRole } from "./authTypes";

export type Permission =
  | "*"
  | "clinical:run"
  | "clinical:override"
  | "clinical:view"
  | "view:analytics"
  | "view:dashboard"
  | "billing:view"
  | "billing:manage"
  | "tenant:manage"
  | "user:manage"
  | "ehr:read"
  | "ehr:write"
  | "deployment:manage"
  | "audit:view";

const ROLE_PERMISSIONS: Record<AuthRole, Permission[]> = {
  admin: ["*"],
  physician: [
    "clinical:run",
    "clinical:override",
    "clinical:view",
    "view:analytics",
    "view:dashboard",
    "ehr:read",
    "ehr:write",
    "audit:view",
  ],
  reviewer: [
    "clinical:view",
    "view:analytics",
    "view:dashboard",
    "audit:view",
  ],
  staff: [
    "billing:view",
    "billing:manage",
    "view:dashboard",
    "audit:view",
  ],
};

export class RBACService {
  can(role: AuthRole, action: Permission): boolean {
    const perms = ROLE_PERMISSIONS[role] ?? [];
    return perms.includes("*") || perms.includes(action);
  }

  canAll(role: AuthRole, actions: Permission[]): boolean {
    return actions.every((action) => this.can(role, action));
  }

  getPermissions(role: AuthRole): Permission[] {
    return ROLE_PERMISSIONS[role] ?? [];
  }
}

// server/auth/authzMiddleware.ts
import { Request, Response, NextFunction } from "express";
import { verifyAccessToken, isJwtError, isTokenExpiredError } from "./unifiedAuth";
import { AccessTokenClaims } from "./authTypes";
import { rbacService, Permission } from "./rbacService";
import { authSessionService } from "./authSessionService";

declare global {
  namespace Express {
    interface Request {
      user?: AccessTokenClaims;
    }
  }
}

function extractBearerToken(req: Request): string | null {
  const auth = req.headers.authorization;
  if (!auth?.startsWith("Bearer ")) return null;
  return auth.slice("Bearer ".length);
}

export async function requireAuth(
  req: Request,
  res: Response,
  next: NextFunction
): Promise<void> {
  const token = extractBearerToken(req);
  if (!token) {
    res.status(401).json({ error: "Missing bearer token" });
    return;
  }

  let claims: AccessTokenClaims;
  try {
    claims = verifyAccessToken(token);
  } catch (err) {
    if (isTokenExpiredError(err)) {
      res.status(401).json({ error: "Token expired" });
      return;
    }
    if (isJwtError(err)) {
      res.status(401).json({ error: "Invalid token" });
      return;
    }
    res.status(401).json({ error: "Authentication failed" });
    return;
  }

  const active = await authSessionService.isTokenActive({
    userId: claims.sub,
    tokenVersion: claims.tokenVersion,
    jti: claims.jti,
  });

  if (!active) {
    res.status(401).json({ error: "Session revoked" });
    return;
  }

  req.user = claims;
  next();
}

export function requirePermission(permission: Permission) {
  return (req: Request, res: Response, next: NextFunction): void => {
    if (!req.user) {
      res.status(401).json({ error: "Authentication required" });
      return;
    }

    if (!rbacService.can(req.user.role, permission)) {
      res.status(403).json({ error: "Insufficient permissions" });
      return;
    }

    next();
  };
}

import { db } from "../db";
import { revokedTokens, users } from "../../shared/schema";
import { eq } from "drizzle-orm";

export const authSessionService = {
  async isTokenActive({
    userId,
    tokenVersion,
    jti,
  }: {
    userId: string;
    tokenVersion: number;
    jti: string;
  }): Promise<boolean> {
    const userRows = await db
      .select({ tokenVersion: users.tokenVersion })
      .from(users)
      .where(eq(users.id, userId))
      .limit(1);

    if (!userRows.length) return false;
    if (userRows[0].tokenVersion !== tokenVersion) return false;

    const revokedRows = await db
      .select({ jti: revokedTokens.jti })
      .from(revokedTokens)
      .where(eq(revokedTokens.jti, jti))
      .limit(1);

    return revokedRows.length === 0;
  },

  async revokeToken({
    jti,
    userId,
    reason,
  }: {
    jti: string;
    userId: string;
    reason?: string;
  }): Promise<void> {
    await db.insert(revokedTokens).values({
      jti,
      userId,
      reason: reason ?? null,
    });
  },

  async revokeAllUserSessions(userId: string): Promise<void> {
    const row = await db
      .select({ tokenVersion: users.tokenVersion })
      .from(users)
      .where(eq(users.id, userId))
      .limit(1);

    if (!row.length) return;

    await db
      .update(users)
      .set({ tokenVersion: row[0].tokenVersion + 1 })
      .where(eq(users.id, userId));
  },
};
