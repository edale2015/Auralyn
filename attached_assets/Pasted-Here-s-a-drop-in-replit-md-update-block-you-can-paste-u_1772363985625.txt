If you want this to stay forever-up-to-date without manual edits, here’s a tiny script you can add later to generate that section directly from COMPLAINT_REGISTRY.csv (so replit.md becomes a computed artifact):

// scripts/print-inventory.ts
import fs from "node:fs";

const csvPath = "data/COMPLAINT_REGISTRY.csv"; // adjust to your actual path
const csv = fs.readFileSync(csvPath, "utf8").trim().split("\n");
const header = csv[0].split(",");
const idxSystem = header.indexOf("System");
const idxSlug = header.indexOf("Complaint_Slug");

const map = new Map<string, string[]>();
for (const line of csv.slice(1)) {
  const cols = line.split(/,(?=(?:[^"]*"[^"]*")*[^"]*$)/); // naive CSV split w/ quotes
  const system = (cols[idxSystem] ?? "").replaceAll('"', "").trim();
  const slug = (cols[idxSlug] ?? "").replaceAll('"', "").trim();
  if (!system || !slug) continue;
  if (!map.has(system)) map.set(system, []);
  map.get(system)!.push(slug);
}

const systems = Array.from(map.keys()).sort();
console.log("## 📦 Consolidated System Inventory (Complaint Pipelines)\n");
console.log("| System | Complaint Pipelines (slugs) |");
console.log("|---|---|");
for (const sys of systems) {
  const slugs = map.get(sys)!.sort().join(", ");
  console.log(`| **${sys}** | ${slugs} |`);
}

(If your repo paths/CSV headers differ, swap those two strings and it’ll print the exact table above automatically.)
