2) Next infrastructure upgrade: move files to Cloud Storage (when you’re ready)

You already documented the risk. When you’re ready, the migration is tiny:

Keep the same API contract

POST /api/intake/:token/upload returns { fileId, ... }

GET /api/file/:fileId still works

Only change the backend storage implementation

Write uploaded bytes to Cloud Storage

Store metadata:

storageMode: "gcs"

gcsPath: "gs://bucket/..."

contentType

For downloads:

either stream from Cloud Storage through your server

or generate signed URLs (short-lived) if you want to offload bandwidth

This is the one change that makes Firestore + multi-instance truly production-safe.
