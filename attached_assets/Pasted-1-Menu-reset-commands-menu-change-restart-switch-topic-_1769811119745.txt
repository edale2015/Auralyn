Add resend block in webhook (after answersObj parsing, early)
const isLinkResend = ["link", "code", "status"].includes(cmd);

if (isLinkResend) {
  const hasToken = Boolean((encounter as any).intakeToken);
  const hasCode = Boolean((encounter as any).intakeCode);
  const exp = Number((encounter as any).intakeExpiresAt || 0);

  if (hasToken && hasCode && exp && Date.now() < exp) {
    const link = `${BASE_URL}/intake/${(encounter as any).intakeToken}`;
    const minutesLeft = Math.max(0, Math.ceil((exp - Date.now()) / 60000));
    await sendWhatsAppMessage(
      phoneNumber,
      `Here’s your secure intake link:\n${link}\nCode: ${(encounter as any).intakeCode}\n(Expires in ~${minutesLeft} min)`
    );
    return res.status(200).send("ok");
  }

  // Otherwise: reset flow selection menu (or restart)
  const updated = setMenuState(answersObj, { awaitingChoice: true, awaitingOtherText: false });
  await storage.updateEncounter(encounter.id, { answers: JSON.stringify(updated) } as any);
  await sendWhatsAppMessage(phoneNumber, "Your previous link expired. Let’s start again.\n\n" + menuText());
  return res.status(200).send("ok");
}


This is idempotent and avoids “new encounter spam.”

Telegram vs Apple messaging
Telegram

Easy. Telegram bots support inline buttons (keyboards) and callback handlers. It’s straightforward to add a clinician-only Telegram bot for “Approve / ED / Call patient,” or to send physician alerts. Even easier than WhatsApp in some ways.

For patients: Telegram is fine, but since you’ve committed to WhatsApp + web grid, I’d keep Telegram as clinician-only if you add it.

Apple Messages / iPhone messaging (iMessage/SMS)

SMS is plain text, no buttons/grids.

iMessage does not have a universal public bot API like Telegram.

Apple does have Apple Messages for Business, but it’s a separate onboarding process and not as simple as “add a token.”

So: Telegram is easy, Apple messaging is not (compared to WhatsApp).
