4) Security quick note about the provider key

provider-secret-key-2026 is fine for dev, but for anything beyond dev:

Don’t bake it into client bundles (avoid VITE_PROVIDER_KEY in production)

Store it server-side and issue a real session token after login

Rotate the provider key regularly (monthly is plenty)

If you keep it simple for now: at least gate provider endpoints by both:

X-Provider-Key

and IP allowlist (optional, if you have stable office IP)
