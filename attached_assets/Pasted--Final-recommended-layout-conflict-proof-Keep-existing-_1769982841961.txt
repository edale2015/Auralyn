🔥 Two important “gotchas” to fix now (even if tests pass)
1) Firestore requires a composite index for your “latest case by token” query

If your Firestore driver does:

cases.where("token","==",token).orderBy("created_at","desc").limit(1)


Firestore will likely require a composite index. When it fails, Firestore returns a console link to create it.

Create index:

Collection: cases

Fields:

token Asc

created_at Desc

If your code doesn’t handle the missing-index error gracefully, you’ll get a runtime failure in production.
