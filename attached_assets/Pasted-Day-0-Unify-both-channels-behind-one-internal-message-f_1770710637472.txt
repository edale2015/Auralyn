Day 0: Unify both channels behind one internal message format
1) Create a single MessageEvent shape

Both WhatsApp and Telegram handlers should emit the same object:

channel: "whatsapp" | "telegram"

externalUserId (whatsapp number or telegram user id)

chatId (telegram chat id; whatsapp can mirror number)

text

timestamp

messageId

rawSignatureVerified: boolean

optional: media[]

This is what drives your orchestrator and makes logs comparable.

2) One “conversation key”

Define conversationId = channel + ":" + externalUserId (or chatId for Telegram groups if you ever allow them).

Store:

conversationId

caseId/encounterId

routingState

lastQuestionIdAsked

requiredMissing[]

toneProfile

lastNMessages[] (redacted if needed)

Day 2: Telegram intake (recommended “right” way)

Telegram is easy to wire, but there are two common footguns: signature/auth and chat context.

6) Telegram bot setup

Create a bot via BotFather, get TELEGRAM_BOT_TOKEN

Set webhook to: POST /api/webhooks/telegram

Validate inbound requests:

Telegram doesn’t give you a Twilio-style signature by default

Best practice: validate by checking webhook secret token (Telegram supports this) or verify requests come from Telegram IP ranges (less ideal)

At minimum: require a secret header like X-Telegram-Bot-Api-Secret-Token

7) Normalize Telegram updates into MessageEvent

externalUserId = update.message.from.id

chatId = update.message.chat.id

messageId = update.message.message_id

text = update.message.text || caption || ""

Important: Telegram can be group chats. For intake, I’d recommend:

only allow 1:1 chats with the bot, reject groups with a friendly message.

8) Telegram outbound replies

Telegram expects JSON sendMessage calls. Add a channel adapter:

sendReply(conversationId, text)
that routes to Twilio for WhatsApp and to Telegram API for Telegram.

Make it testable exactly how you want: “see what triggered and why”

“Problem patient” handling in chat channels

You’re already detecting friction. For WhatsApp/Telegram intake, add a policy:
