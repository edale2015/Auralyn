If anything fails, here are the 3 usual culprits
1) Firebase Admin Storage bucket not set

Make sure your firebase init uses one of:

admin.initializeApp({ storageBucket: process.env.FIREBASE_STORAGE_BUCKET })
or your code explicitly uses:

admin.storage().bucket(process.env.FIREBASE_STORAGE_BUCKET)

2) Wrong bucket name

For Firebase default buckets it’s typically:

<project-id>.appspot.com

You already have: medicalm-dec9d.appspot.com ✅

3) Permissions

If your service account doesn’t have storage permissions, upload will fail. Most Firebase service accounts do, but if you locked it down, you might see 403s.
Fix: give the service account Storage Object Admin (or equivalent minimal role).
