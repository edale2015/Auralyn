4) Verify validation + auto-rollback triggers on failure

Force validation to fail safely:

A) Run with a validator that always fails
npx tsx scripts/apply-profile-pack-bulk.ts data/complaints/profile_apply_seed.csv --cc sinus_pressure --validate-cmd "exit 1"
echo $?

Expected:

It should create a backup run folder

It should write, then say validation failed

It should say auto-rollback

Exit code should be 1

Then confirm the files match backup after rollback:

# find the newest backup run id
ls -1t server/data/csv/_tx_backups | head -n 1
# compare restored file to its backup
RUN_ID=$(ls -1t server/data/csv/_tx_backups | head -n 1)
diff -u server/data/csv/DX_PRIORITY.csv server/data/csv/_tx_backups/$RUN_ID/DX_PRIORITY.csv.bak | head
diff -u server/data/csv/CLUSTER_SCORING_RULES.csv server/data/csv/_tx_backups/$RUN_ID/CLUSTER_SCORING_RULES.csv.bak | head

Expected: no diff output (files identical).

B) Verify --no-auto-rollback really disables rollback
npx tsx scripts/apply-profile-pack-bulk.ts data/complaints/profile_apply_seed.csv --cc sinus_pressure --validate-cmd "exit 1" --no-auto-rollback
echo $?

Expected:

it prints “auto-rollback disabled”

exit 1

files will remain as written (so only do this when you’re ready to rollback manually using --rollback)
