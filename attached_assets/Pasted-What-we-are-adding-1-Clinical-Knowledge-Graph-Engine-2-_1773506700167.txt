1️⃣ Clinical Knowledge Graph Engine

This becomes the central brain database.

Instead of rules like:

if cough + fever → pneumonia

we store relationships like:

symptom → diagnoses
diagnosis → treatments
diagnosis → tests
diagnosis → red flags

This is the same concept used by Ada Health and Isabel.

Create file
server/core/knowledgeGraphEngine.ts
type NodeType =
  | "symptom"
  | "diagnosis"
  | "test"
  | "treatment"
  | "red_flag"

interface GraphEdge {

  from: string

  to: string

  weight: number

  type: NodeType
}

class KnowledgeGraphEngine {

  private edges: GraphEdge[] = []

  constructor(edges: GraphEdge[]) {

    this.edges = edges
  }

  findDiagnosesFromSymptoms(symptoms: string[]) {

    const scores: Record<string, number> = {}

    for (const s of symptoms) {

      const related = this.edges.filter(
        e => e.from === s && e.type === "diagnosis"
      )

      for (const r of related) {

        scores[r.to] = (scores[r.to] || 0) + r.weight
      }
    }

    return Object.entries(scores)
      .map(([dx, score]) => ({ diagnosis: dx, score }))
      .sort((a,b)=>b.score-a.score)
  }

  getTestsForDiagnosis(dx:string){

    return this.edges
      .filter(e=>e.from===dx && e.type==="test")
      .map(e=>e.to)
  }

  getTreatmentsForDiagnosis(dx:string){

    return this.edges
      .filter(e=>e.from===dx && e.type==="treatment")
      .map(e=>e.to)
  }

  getRedFlags(dx:string){

    return this.edges
      .filter(e=>e.from===dx && e.type==="red_flag")
      .map(e=>e.to)
  }
}

export default KnowledgeGraphEngine

3️⃣ Diagnostic Evidence Engine

This layer merges

knowledge graph

Bayesian engine

case similarity

Create

server/core/diagnosticEvidenceEngine.ts
import KnowledgeGraphEngine from "./knowledgeGraphEngine"
import { clinicalEdges } from "../data/clinicalKnowledgeGraph"

const graph = new KnowledgeGraphEngine(clinicalEdges)

export function diagnosticEvidenceEngine(symptoms:string[]){

  const graphResults = graph.findDiagnosesFromSymptoms(symptoms)

  return graphResults.slice(0,10)
}
4️⃣ Symptom Normalization Engine

Patients say:

my chest feels tight

chest_pain

const synonymMap:Record<string,string>={

  "chest tightness":"chest_pain",
  "pressure in chest":"chest_pain",

  "burning urination":"dysuria",
  "pain when peeing":"dysuria",

  "throwing up":"vomiting"
}

export function normalizeSymptoms(input:string[]){

  return input.map(s=>synonymMap[s] || s)
}
5️⃣ Clinical Safety Guard Engine

This must override reasoning.

Create

server/core/clinicalSafetyGuard.ts
export function safetyGuard(symptoms:string[]){

  if(symptoms.includes("testicular_pain"))
    return "ER_NOW"

  if(symptoms.includes("chest_pain") &&
     symptoms.includes("shortness_of_breath"))
    return "ER_NOW"

  if(symptoms.includes("neurologic_deficit"))
    return "ER_NOW"

  return null
}
6️⃣ Uncertainty / Entropy Engine

Doctors ask more questions when uncertain.

Create

server/core/uncertaintyEngine.ts
export function entropy(scores:number[]){

 let total = scores.reduce((a,b)=>a+b,0)

 if(total===0) return 0

 let e=0

 for(const s of scores){

   const p=s/total

   if(p>0) e-=p*Math.log(p)
 }

 return e
}

If entropy high → ask next question.

9️⃣ Return Precaution Engine

Create

server/core/returnPrecautionEngine.ts
export function generateReturnPrecautions(dx:string){

  const precautions:Record<string,string[]>={

    pneumonia:[
      "worsening shortness of breath",
      "persistent fever",
      "chest pain"
    ],

    UTI:[
      "fever",
      "back pain",
      "vomiting"
    ]
  }

  return precautions[dx] || []
}
