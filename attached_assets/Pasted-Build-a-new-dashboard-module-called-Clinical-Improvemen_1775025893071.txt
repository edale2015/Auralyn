Part 2. Trial Branch Sandbox

Add an experiment sandbox UI where I can:

clone an existing pathway or complaint tree
edit one branch or one rule
save as trial version
run 1k / 10k / 50k cases
compare trial vs production
see expected impact before promotion
promote the better branch if approved

The dashboard should support examples like:

for headache: add BP check
for women of childbearing age with headache: consider pregnancy
for asthma: ask albuterol frequency
and then run thousands of cases to see if the change helps
Part 3. 5 proactive intelligence features

Add these five continuous-review features:

1. Branch fragility scanner

Continuously identify branches where small rule changes create large output swings.
Show:

complaint
branch
sensitivity score
why it is fragile
2. Cross-complaint modifier coverage audit

Continuously scan modifiers such as:

pregnancy
CHF
CKD
anticoagulation
penicillin allergy
Show where they are missing across complaints/systems.
3. Outcome-weighted recommendation ranking

Rank suggestions not just by confidence, but by:

expected safety gain
expected cost reduction
expected reduction in false reassurance
expected admission impact
4. Silent-failure detector

Detect KB rules that are rarely or never triggered in simulation or live operation.
Show:

rule ID
complaint
domain
last triggered
suspicion reason
5. Automatic trial-branch promoter

Support sandbox experiments where a branch can be promoted only after:

simulation passes
no safety regression
approval is granted
audit trail is written

Part 6. Audit and governance

Every experiment and promotion must be auditable.
Log:

who made the trial change
old branch
new branch
simulation results
approval decision
promotion time
rollback option

No high-risk branch should auto-promote without approval.

Part 7. Acceptance criteria

I should be able to:

review a complaint pathway
clone it
add or edit a branch
run thousands of cases
compare production vs trial
see whether safety and outcomes improve
approve and promote the better branch
inspect fragility, modifier gaps, and unused rules from the same dashboard

Goal:
Make the system continuously self-reviewing and experimentally improvable from the dashboard, without needing code edits for normal clinical pathway refinement.
