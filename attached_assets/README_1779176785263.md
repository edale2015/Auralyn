# Auralyn Skills — Installation & Usage

Six Claude Code skills that encode the recurring patterns, conventions,
and known failure modes for the Auralyn project. Once installed, an AI
coding assistant automatically loads the relevant skill when it
encounters a matching context — no need to re-explain Auralyn's
architecture, your session-plan format, or the no-fudging rules every
session.

## What's in this package

| Skill | When it loads | What it teaches |
|-------|---------------|-----------------|
| `auralyn-session-plan` | When asked for tasks, plans, "next steps" | The exact T###/V###/F### format Replit's agent acts on |
| `auralyn-no-fudging` | When finishing tasks or writing status reports | The 7 specific failure modes observed and how to avoid them |
| `auralyn-context-engineering` | When touching `server/context/` or the pipeline | The tiered context model, role contracts, what already exists |
| `auralyn-clinical-safety` | When modifying clinical reasoning or PHI handling | The 8 non-negotiable safety invariants |
| `auralyn-kb-architecture` | When working with `kb_master_rules` or the rule engine | Schema, rule types, the WHEN_EXPR gotcha, config loading flow |
| `auralyn-regulatory` | When discussing FDA, 510(k), HIPAA, RLS | Posture summary, RLS patterns, doc-impact triggers |

## How skills work (short version)

Each skill is a folder with a `SKILL.md` file. The YAML frontmatter at
the top defines the skill's name and the trigger description. The body
contains the actual instructions.

At session start, the AI assistant reads only the metadata (a few dozen
tokens per skill). When your conversation matches a skill's trigger
description, the assistant loads the full body. This is called
"progressive disclosure" — many skills can be installed without
burning context.

## Installation — Option A: Claude Code (recommended)

Claude Code is Anthropic's command-line coding agent. It natively
supports the Skills format. Skills installed in `.claude/skills/` are
loaded automatically.

1. **Install Claude Code** if you don't have it:
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```

2. **Copy this folder** into your Auralyn project at
   `.claude/skills/auralyn/`:
   ```bash
   # From your project root
   mkdir -p .claude/skills
   cp -r /path/to/auralyn-skills/* .claude/skills/
   ```

3. **Verify they're loaded** — start Claude Code in your project and
   ask: *"What Auralyn skills do you have available?"* It should list
   all six.

4. **Use them** — just work normally. The skills load automatically
   when relevant. You don't have to invoke them by name.

## Installation — Option B: Replit Agent

Replit's agent may not natively honor the `.claude/skills/` format
(check current Replit documentation). For Replit, treat these files as
context documents you paste into the chat at the start of a session.

1. **Copy the relevant skill's SKILL.md content** into Replit's chat
   when starting a session on that topic.
   - Working on the context module? Paste `auralyn-context-engineering`.
   - Writing a session plan? Paste `auralyn-session-plan` AND
     `auralyn-no-fudging`.
   - Touching clinical reasoning? Paste `auralyn-clinical-safety`.

2. **Or:** put a single `AURALYN_CONTEXT.md` file at your project root
   that concatenates the most-frequently-needed skills (probably
   `no-fudging` + `session-plan` + `clinical-safety`). Tell the agent
   to read this file at session start.

## Installation — Option C: Both

Recommended if you want maximum coverage:

1. Install Claude Code with the skills for terminal-based work.
2. Also maintain an `AURALYN_CONTEXT.md` summary file for Replit Agent
   sessions.

## Maintenance

These skills encode the project state as of their creation. When the
project changes materially (new modules, new contracts, new failure
modes), update the relevant skill.

The `description:` line in the YAML frontmatter is the most important
field — it determines when the skill loads. If you find a skill loading
too often or not enough, edit that line first.

Keep each `SKILL.md` under 500 lines per the Skills best practices. If
a skill grows beyond that, split detail into `reference.md` or
`templates/` files and reference them from `SKILL.md`.

## What these skills will NOT fix

- They won't make the AI agent more capable than it is. Skills shape
  behavior; they don't add reasoning power.
- They won't catch every failure. The `no-fudging` skill encodes the
  patterns observed so far. New failure modes need new entries.
- They don't replace verification. The skills tell the agent how to
  verify; the human still has to read the output.

## What they will do

- Stop re-explaining the same architecture every session.
- Stop the "declare done without verifying" pattern (most of it).
- Stop the "detour to verify already-completed work" pattern.
- Keep the session-plan format consistent so Replit's agent reliably
  acts on it.
- Surface clinical safety concerns proactively when relevant.

If you observe a new failure pattern, add it to `auralyn-no-fudging`
and the next session benefits.
