## T010 — Plug in `AgentArtifactBus` between agents

**Status:** ⏳ Not started

**Files to modify:**
- The pipeline orchestrator (likely `server/clinical/pipelineOrchestrator.ts`
  or `server/clinical/agents/index.ts` — Replit assistant: locate the file
  that calls agents in sequence)

**Acceptance criteria:**
- A single `AgentArtifactBus` is created per encounter and held alongside
  the `ClinicalContextManager`
- After each agent step, the agent calls `bus.publish(role, artifact)` for
  every new artifact it produced
- Before each agent step, the agent receives `bus.readFor(role)` and uses
  ONLY those artifacts (no longer reads the full prior transcript)
- The billing agent in particular must see a strictly smaller artifact set
  than the differential agent — verify this in the verification step
- Contract violations (an agent trying to publish a type it doesn't own)
  should THROW, not be silently swallowed — let it crash in dev so we find
  these

## T014 — Unit tests for `ContextCompactor` safety properties

**Status:** ⏳ Not started

**Files to create:**
- `src/context/__tests__/ContextCompactor.test.ts`

**Acceptance criteria:** the test suite covers AT MINIMUM these cases:
1. Compactor never modifies immutables (red flags, allergies, vitals,
   chief complaint) regardless of input
2. Stale + low-likelihood differential entries become `ruled_out` artifacts
3. Stale pending questions become `uncertainty` artifacts (not silently
   dropped)
4. High-likelihood entries are NEVER dropped even if not recently updated
5. Most recent N answered questions are preserved verbatim
6. The compactor is deterministic — same input produces same output across
   100 runs (no Math.random or Date.now affecting outputs in ways that
   change content)
7. A `compaction_summary` artifact is emitted whenever any summarization
   occurs

**Verification:**
```bash
npm test -- src/context/__tests__/ContextCompactor.test.ts
# expect: all tests pass
```
