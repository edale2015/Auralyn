3) Urgent-care HTN “spot intervention” layer (safe escalation basics)

You want: catch bad patterns, do minimal safe actions, bridge until PCP.

A. Triage first (always)

Your rules should separate:

Possible hypertensive emergency → ER

neuro deficits, chest pain, acute SOB/pulm edema, vision loss, confusion, seizure

pregnancy with severe HTN symptoms

suspected aortic catastrophe

Severe asymptomatic HTN → urgent follow-up + limited workup + cautious approach
(UC practice varies; keep your system conservative.)

B. UC-friendly HTN interventions you can safely standardize

Your “spot intervention” output can include:

Confirm correct BP technique (repeat BP, cuff size, rest)

Assess meds + adherence (missed doses, affordability, side effects)

Check for secondary contributors

NSAIDs, decongestants, stimulants, steroids, cocaine, heavy alcohol, OSA risk

Basic labs only when appropriate (especially if adding/adjusting meds)

BMP/CMP (K/Cr), urine albumin/Cr if possible

Medication fill-gap (only if clinician chooses)

common “tripod” concept: ACE/ARB + CCB + thiazide

Home BP plan

2 readings AM/PM for 3–7 days, average, bring log

Clear safety net

return/ER triggers

C. Add these HTN escalation rules (table-style)

You can implement as MED_CONDITION_INTELLIGENCE_RULES rows like:

HTN_ESC_001 (BP confirmation + red flag screen)

HTN_ESC_002 (med-gap detection: on 1 agent only, uncontrolled)

HTN_ESC_003 (resistant HTN screen: 3 meds including diuretic and still high)

HTN_ESC_004 (OSA contributor flag)

HTN_ESC_005 (UC bridge plan: labs + short refill + follow-up)

And for meds, add adjacency triggers:

valsartan/losartan/olmesartan

lisinopril

amlodipine

HCTZ/chlorthalidone (if you want)

spironolactone (resistant HTN context)

4) Urgent-care DM “spot intervention” layer (safe escalation basics)

Goal: detect danger, prevent complications, bridge access gaps.

A. Triage first

Send to ER if you suspect:

DKA/HHS features: altered mental status, severe dehydration, Kussmaul, persistent vomiting, severe abdominal pain

very high glucose with symptoms (you define threshold in your clinic protocols)

severe hypoglycemia not quickly reversible

infection + unstable vitals in DM/immunosuppressed

B. UC-friendly DM interventions you can standardize

Identify DM type and meds

Check “crisis questions”

polyuria/polydipsia, vomiting, abdominal pain, confusion, fruity breath

Basic point-of-care options (if available)

fingerstick glucose

urine ketones (especially if symptomatic or type 1)

Bridge plan

refill essential meds when safe

sick-day rules education

hypoglycemia plan

Care gap prompts

last A1c, eye exam, microalbumin, statin eligibility, ACE/ARB if albuminuria/HTN

Medication selection guidance (non-prescriptive)

weight/CHF/CKD favor GLP-1/SGLT2 (if eligible and accessible)

hypoglycemia risk meds flagged (sulfonylureas, insulin)

C. Add DM escalation “regimen logic” rules (UC-safe)

Rather than trying to prescribe full algorithms, your system should output:

“If uncontrolled on X, consider adding Y” as a clinician suggestion

“If hypoglycemia on sulfonylurea, consider switching strategy”

“If CHF/CKD, consider SGLT2 class discussion”

Add rules:

DM_ESC_001 (danger screen: DKA/HHS/hypo)

DM_ESC_002 (care gaps + essential labs)

DM_ESC_003 (med class fit: weight/CHF/CKD)

DM_ESC_004 (GLP-1 side effects routing already built)

DM_ESC_005 (UC bridge/refill plan + follow-up)
