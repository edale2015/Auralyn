A) Where complaint mapping typically breaks

Let’s trace the chain:

raw text
  ↓
parseComplaint()
  ↓
ComplaintCode (string)
  ↓
ComplaintRegistryEntry (CSV/DB)
  ↓
ComplaintConfig
  ↓
Node Runner
1. String-based mapping (fragile glue)

You likely have:

const parsed = parseComplaint(text);
const config = loadComplaintConfig(parsed.primary);
Risks:
typo mismatch (sore_throat vs soreThroat)
config exists but disabled
config missing → crash or silent fallback
aliasing not handled (“UTI” vs “urinary infection”)

👉 No canonical mapping layer

2. Registry is passive, not enforced
ComplaintRegistryEntry {
  ccId: string;
  scoringModule: string;
  enabled: boolean;
}
Risks:
enabled = false but still used
missing entry → fallback or crash
wrong scoringModule string → runtime error later

👉 Registry is data, not a gatekeeper

3. No validation between parser ↔ registry

Parser produces:

"sore_throat"

But registry may contain:

"soreThroat"

→ mismatch → config not found

👉 No contract enforcement

4. No fallback strategy

If mapping fails:

return undefined → flow breaks
default complaint → unsafe misrouting
throw → bad UX

👉 Needs explicit fallback policy

5. No versioning awareness

If registry updates:

parser unchanged
mapping breaks silently
B) What a production-safe mapping layer should do

You want a single authoritative resolver:

ParsedComplaint → ComplaintResolution

With guarantees:

canonical ID
validated existence
enabled check
scorer exists
config load succeeds
fallback defined

2. Resolver layer (THIS is the missing piece)
// server/clinical/complaintResolver.ts

export interface ComplaintResolution {
  ccId: string;
  source: "primary" | "alias" | "fallback";
  confidence: "high" | "low";
}

export async function resolveComplaint(
  parsed: ParsedComplaint | undefined
): Promise<ComplaintResolution> {

  if (!parsed) {
    return fallbackComplaint("no_parse");
  }

  const registry = await getComplaintRegistry(); // DB or cached

  // ── Exact match ─────────────────────────────
  const direct = registry.find(r => r.ccId === parsed.primary);

  if (direct && direct.enabled) {
    return {
      ccId: direct.ccId,
      source: "primary",
      confidence: parsed.confidence,
    };
  }

  // ── Alias match ─────────────────────────────
  for (const entry of registry) {
    if (entry.aliases.includes(parsed.primary) && entry.enabled) {
      return {
        ccId: entry.ccId,
        source: "alias",
        confidence: "low",
      };
    }
  }

  // ── Fallback ────────────────────────────────
  return fallbackComplaint(parsed.primary);
}

function fallbackComplaint(reason: string): ComplaintResolution {
  return {
    ccId: "general_symptom",
    source: "fallback",
    confidence: "low",
  };
}
3. Enforce registry integrity at load time
function validateRegistry(registry: ComplaintRegistryEntry[]) {
  const ids = new Set();

  for (const entry of registry) {
    if (!entry.ccId) {
      throw new Error("Missing ccId in registry");
    }

    if (ids.has(entry.ccId)) {
      throw new Error(`Duplicate complaint id: ${entry.ccId}`);
    }

    ids.add(entry.ccId);

    if (!entry.scoringModule) {
      throw new Error(`Missing scoringModule for ${entry.ccId}`);
    }
  }
}
4. Safe config loading integration
export async function getResolvedComplaintConfig(
  parsed: ParsedComplaint | undefined
): Promise<ComplaintConfig> {

  const resolution = await resolveComplaint(parsed);

  try {
    return await loadComplaintConfig(resolution.ccId);
  } catch (err) {
    console.error("[ComplaintMapping] Config load failed:", err);

    // fallback to safe default
    return await loadComplaintConfig("general_symptom");
  }
}
